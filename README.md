# Proyecto 2

# Portfolio

## Descripción del Proyecto
Este proyecto he elegido hacer  el  portafolio personal del cliente , lo he diseñado para verificar mis  habilidades en desarrollo web y diseño UI/UX. Utiliza HTML5 y CSS3, aplicando principios de diseño responsive y accesibilidad para garantizar una experiencia óptima para todos los usuarios. Este portafolio no solo sirve como escaparate del trabajo profesional de Jose, sino también como una herramienta de contacto para futuras colaboraciones profesionales.

## Tecnologías Utilizadas

Este proyecto lo he construido exclusivamente con **HTML5** y **CSS3**, aprovechando las capacidades modernas de ambos para crear un sitio web responsivo y estéticamente agradable. A continuación se detallan los aspectos clave:

### HTML5
- **Uso de Etiquetas Semánticas**: Implementación de `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, y `<footer>` para estructurar el contenido de forma lógica y accesible.
- **Formularios HTML5**: Utilización de tipos de entrada como `email` para mejorar la validación de formularios.

### CSS3
- **Flexbox y Grid**: Empleados para crear layouts responsivos que se adaptan a diferentes tamaños de pantalla.
- **Transiciones y Animaciones**: Aplicadas para mejorar la interactividad y el atractivo visual del sitio.
- **Variables CSS**: Usadas para mantener la coherencia en los estilos a lo largo del proyecto.

Estas tecnologías aseguran que el sitio no solo sea funcional y atractivo, sino también accesible y compatible con todos los navegadores modernos.


## Estructura de directorios del Proyecto
- `index.html`: Página principal del sitio.
- `about.html`: Información sobre el creador o el propósito del proyecto.
- `portfolio.htm`: Contiene la informacion sobre los proyectos donde ha trabajado
- `contacto.html`: La página donde puede contactar con el usuario del portfolio.

- `assets/`: Directorio para imágenes, fuentes y otros recursos estáticos.

## Implementación

### HTML

En el desarrollo de las páginas del sitio, he priorizado el uso de HTML semántico para mejorar tanto la accesibilidad como la estructura del contenido, facilitando su comprensión tanto por los usuarios como por los motores de búsqueda. Aquí algunos ejemplos de cómo he aplicado etiquetas semánticas:

- `<header>`: Utilizada para envolver el contenido de la cabecera principal de cada página, incluyendo el menú de navegación y el logotipo del sitio.
- `<nav>`: Dentro del `<header>`, para identificar claramente la sección de navegación principal del sitio.
- `<main>`: Representa el contenido principal de cada página, asegurando que este sea el foco principal para los lectores de pantalla y los motores de búsqueda.
- `<section>`: Usada para definir secciones dentro del `<main>`, cada una representando un grupo temático distinto de contenido.
- `<article>`: Para artículos independientes que podrían ser distribuidos fuera del contexto del sitio, como posts de blog o noticias.
- `<footer>`: Contiene información sobre los derechos de autor, enlaces a políticas de privacidad y datos de contacto.

### CSS

Para estilizar el sitio, he empleado varias técnicas avanzadas de CSS que mejoran la visualización y la interactividad del sitio:

- **Flexbox**: Utilizado para diseñar un layout flexible y responsivo, facilitando la distribución de espacio entre elementos de una manera eficiente. Por ejemplo, se usa en el diseño del header y en la barra de navegación para alinear los elementos horizontalmente.
- **Grid**: Empleado para estructuras más complejas, como la galería de proyectos en la página de portfolio, donde cada proyecto se muestra en su propia celda dentro de un grid sistemático.
- **Animaciones**: Implementadas para mejorar la interactividad y el atractivo visual del sitio. Por ejemplo, animaciones suaves en los botones y enlaces para indicar interactividad.
- **Media Queries**: Para garantizar que el sitio sea completamente responsivo, las media queries se han utilizado para ajustar los estilos en diferentes tamaños de pantalla, asegurando que el sitio se vea bien en dispositivos móviles, tabletas y escritorios.

Cada uno de estos elementos de CSS no solo mejora la experiencia del usuario sino que también apoya la accesibilidad y la usabilidad del sitio.

## Validación del Proyecto

### Validación del HTML y CSS

Este proyecto ha pasado por un proceso exhaustivo de validación para asegurar la calidad y el cumplimiento de los estándares web. Aquí están los detalles sobre cómo se validaron los archivos HTML y CSS:

#### Herramientas Utilizadas:
- **HTML**: Validación realizada utilizando el [W3C Markup Validation Service](https://validator.w3.org/), que ayuda a verificar que las páginas web sigan los estándares definidos por el World Wide Web Consortium (W3C).
- **CSS**: Validación realizada con el [W3C CSS Validation Service](https://jigsaw.w3.org/css-validator/), que comprueba la corrección del CSS según los estándares de CSS.

#### Proceso de Validación:
1. **HTML**:
   - Cada archivo HTML ha sido subido al validador del W3C a través de la interfaz de usuario proporcionada en su sitio web.
   - Se corrigieron todos los errores identificados por el validador para asegurar que el HTML esté completamente conforme a los estándares.

2. **CSS**:
   - Los archivos CSS fueron validados subiendo el código fuente al validador de CSS del W3C.
   - Se ajustaron las declaraciones de CSS según las recomendaciones del validador para eliminar advertencias y errores.

#### Resultados:
- Todos los archivos HTML y CSS han sido validados sin errores, asegurando que el código sea limpio y esté bien estructurado.
- La validación ha confirmado que el sitio cumple con las mejores prácticas de desarrollo web, lo que contribuye a su rendimiento y accesibilidad.

### Importancia de la Validación
La validación es un paso crucial en el desarrollo web para:
- Asegurar la compatibilidad entre diferentes navegadores y dispositivos.
- Mejorar la accesibilidad para todos los usuarios.
- Mantener el código limpio y fácilmente mantenible.

Esta práctica no solo mejora la calidad del sitio web, sino que también asegura una experiencia de usuario coherente y accesible para todos los visitantes.


## Accesibilidad

La accesibilidad es una prioridad en este proyecto, y se han implementado varias medidas para asegurar que el sitio web sea usable por el mayor número posible de personas, incluyendo aquellas con discapacidades visuales, auditivas, motoras o cognitivas. Aquí se detallan algunas de las estrategias adoptadas:

- **Etiquetas Semánticas de HTML5**: Se han utilizado etiquetas semánticas que ayudan a los lectores de pantalla a entender la estructura del contenido, mejorando la navegación por el sitio.
- **Contraste Adecuado**: Se ha cuidado el contraste de colores para que haya suficiente diferencia entre el color de fondo y el texto, cumpliendo con las pautas WCAG 2.1 AA para la accesibilidad visual.
- **Navegación por Teclado**: El sitio es completamente navegable utilizando solamente un teclado, permitiendo a los usuarios con limitaciones motoras acceder a todas las funcionalidades.
- **Etiquetas `alt` en Imágenes**: Todas las imágenes incluyen atributos `alt` descriptivos que proporcionan equivalencias textuales de la información visual, esencial para usuarios con discapacidades visuales.
- **Tamaños de Fuente Ajustables**: Los estilos CSS están diseñados para permitir el ajuste de tamaño de fuente sin romper la disposición de la página, facilitando la lectura para usuarios con dificultades visuales.

Estas medidas no solo mejoran la experiencia de usuarios con discapacidades, sino que también refuerzan la usabilidad general del sitio para todos los visitantes.

## Proyecto 3

# Proyecto de Testimonios Interactivos

Este proyecto tiene como objetivo crear una página web para mostrar testimonios de clientes con funcionalidad interactiva utilizando **JavaScript**, **HTML** y **CSS**. Los testimonios se muestran de 3 en 3 y permiten al usuario agregar, navegar entre ellos y verlos de manera organizada.

## Estructura del Proyecto

### HTML

- Un contenedor para los testimonios.
- Botones para navegar entre los testimonios ("Anterior" y "Siguiente").
- Un formulario para agregar nuevos testimonios.

### CSS

- Utiliza **Flexbox** para el diseño de los testimonios y los botones de navegación.
- El formulario de testimonios está centrado en la página con un ancho del **40%**.

### JavaScript

- Gestión dinámica de los testimonios (mostrar, agregar, navegar entre ellos).
- Se usan **event listeners** para manejar interacciones del usuario.

---

## ¿Cómo lo hice?

### Uso de JavaScript (ES6)

1. **Declaración de variables:**
    Usé `let` y `const` para declarar las variables. `let` se usa para las variables cuyo valor puede cambiar, como el índice de los testimonios, y `const` para aquellos que no cambian, como el array de testimonios.

    ```javascript
    const testimonials = [ ... ];
    let currentIndex = 0;
    ```

2. **Funciones de flecha:**
    Implementé **arrow functions** para que el código fuera más conciso y para manejar mejor el contexto de `this`, especialmente al trabajar con `addEventListener`.

    ```javascript
    const createTestimonial = (testimonial) => { ... };
    ```

3. **Métodos de array:**
    Usé algunos métodos de **array** como `map`, `filter` y `reduce` para manejar los testimonios de forma eficiente. Aunque en este caso no se usan explícitamente, es algo que se podría añadir más adelante si se necesitara hacer alguna manipulación avanzada de los datos.

4. **Destructuring y Spread Operators:**
    Aunque no he usado **destructuring** ni **spread operators** en este proyecto, podría implementarlos si algún día quiero simplificar la manipulación de los datos o realizar operaciones más complejas.

---

### Manipulación del DOM

1. **Selección de elementos:**
    Para seleccionar los elementos en el DOM, utilicé `document.querySelector()` en lugar de `getElementById()` para obtener más flexibilidad y simplicidad. 

    ```javascript
    const testimonialsContainer = document.querySelector('#testimonials__container');
    const prevButton = document.querySelector('#prevButton');
    const nextButton = document.querySelector('#nextButton');
    ```

2. **Creación y manipulación de elementos:**
    Usé `document.createElement()` para crear elementos dinámicamente, como los testimonios y el formulario. Después, los añadí al DOM usando `appendChild()`.

    ```javascript
    const article = document.createElement('article');
    article.className = 'testimonio';
    ```

3. **Eliminación de elementos:**
    Aunque no implementé la eliminación de testimonios directamente, se podría usar `removeChild()` o `remove()` si decidiera permitir que los usuarios eliminen testimonios.

---

### Funcionalidades Interactivas

1. **Galería interactiva:**
    Los testimonios se muestran de 3 en 3 con la posibilidad de navegar entre ellos usando los botones **"Anterior"** y **"Siguiente"**. Para hacerlo, mantuve un índice actual que cambia según el botón que el usuario presiona.

    ```javascript
    prevButton.addEventListener('click', () => { ... });
    nextButton.addEventListener('click', () => { ... });
    ```

2. **Formulario con validación dinámica:**
    El formulario permite agregar nuevos testimonios. Al momento de enviar el formulario, se valida que tanto el texto del testimonio como el autor estén presentes. 

    ```javascript
    const form = document.createElement('form');
    form.onsubmit = (event) => { ... };
    ```

3. **Sistema de filtros:**
    Aunque no lo implementé de forma explícita, podría agregar filtros para que el usuario pueda buscar testimonios por autor o por palabras clave. Esto se podría hacer con clases CSS para ocultar o mostrar testimonios en función de los criterios de búsqueda.

4. **Botones de navegación:**
    Los botones de **"Anterior"** y **"Siguiente"** están en el centro de la página y permiten navegar entre los testimonios. Esto proporciona una experiencia de usuario fluida.

    ```javascript
    prevButton.addEventListener('click', () => { ... });
    nextButton.addEventListener('click', () => { ... });
    ```

---

### Uso de `classList.add`, `classList.remove`, y `classList.toggle`

1. **Estilos dinámicos:**
    Utilizo `classList.add` para agregar clases de estilo a los elementos creados dinámicamente, y `classList.remove` para eliminarlas si es necesario. Si en el futuro quiero implementar cambios visuales adicionales, podría usar `classList.toggle` para alternar entre clases.

---

### Separación de Lógica y Presentación

1. **Lógica de negocio:**
    La lógica que maneja los testimonios (agregar, mostrar, navegar) está separada de la presentación (estilos CSS). Esto facilita el mantenimiento del proyecto.

    ```javascript
    const updateTestimonio = () => { ... };
    const addTestimonial = (text, author) => { ... };
    ```

2. **Código organizado:**
    He organizado el código de forma que sea claro y comprensible. Cada función tiene un propósito específico y está separada de las demás.

---

He añadido comentarios en las funciones clave para que el código sea fácil de seguir. Esto ayuda a otros desarrolladores (o a mí mismo en el futuro) a entender rápidamente lo que hace cada parte del código.
